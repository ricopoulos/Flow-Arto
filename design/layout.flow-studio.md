## üé® Aesthetic Guidance for Stylist

**Color Direction:**
- Slate (primary: professional, technical, calm)
- Blue (accent: confidence, capability, interactivity)
- White/light gray (surfaces: clean, spacious)
- Minimal color usage‚Äîlet content breathe

**Typography Direction:**
- **Single font system:** Inter (variable weights)
- **Headings:** Inter 700 (bold), slate-700, tight line-height (1.25)
- **Body:** Inter 400 (regular), slate-600, relaxed line-height (1.625)
- **Scale:** 48-60px hero (desktop), 36-48px section headings, 16-18px body

**Spacing:**
- Calm-medium density: 48-96px section padding
- 4px base scale: precise control
- Content max-width: 1280px (wide), 720px (narrow for text)
- Generous whitespace, but not sparse

**Component Patterns:**
- Buttons: 6px radius, 600 weight, 44px height, subtle hover (color shift + minimal shadow)
- Form inputs: 6px radius, 44px height, visible labels, blue focus ring
- Cards: 8px radius, subtle border (#e2e8f0), minimal shadow
- Icons: Line-art, 24-32px, slate or blue

## ‚ôø Accessibility Requirements (WCAG AA+)

- **Contrast:** All text ‚â•7:1 against background (exceeds WCAG AAA)
- **Semantic HTML:** `<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`, proper heading hierarchy (H1 ‚Üí H2 ‚Üí H3)
- **Form Accessibility:**
  - Labels with `for` attribute (not placeholder-only)
  - Error states with ARIA live regions
  - Required fields clearly marked with visual + ARIA
- **Keyboard Navigation:**
  - Visible focus states (blue ring, 3px, 0px offset)
  - Tab order follows visual flow
- **Touch Targets:** Buttons and form inputs ‚â•44px
- **Alt Text:** All icons have descriptive alt text or `aria-label`
- **Motion:** All animations respect `prefers-reduced-motion`
